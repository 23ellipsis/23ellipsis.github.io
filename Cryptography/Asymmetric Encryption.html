Asymmetric Encryption

  1. A mathematical algorithm (being the encryption algorithm) would produce two keys, and they would be a pair.
    • The two keys are mathematically linked.
    • You cannot derive one from the other.
    • If data got encrypted by the first key, the second key would be the only key that can decrypt that data.
    • One of these keys would be a public key and would be shared publicly.
    • The other one would be a private key, and would be kept private.
    • Note that there isn't a functional difference between the public and private key, any could be either.

  2. Both parties would exchange their public keys. This could be done anywhere since the keys are public anyway.

  3. If Person1 with Key1 wanted to send data to Person2 with Key2, he would encrypt the data using Person2's public key.

  4. Now that the data is encrypted with Person2's public key, only his private key can decrypt the data.

  5. And since this model is dependent on the security of the private key, assuming it is kept safe, only Person2 will be able to view the data.
