2.11 SNMP - Simple Network Management Protocol

SNMP "Simple Network Management Protocol" is an application-layer protocol for monitoring and managing network devices on a local area network (LAN) or wide area network (WAN).

The purpose of SNMP is to provide network devices, such as routers, servers and printers, with a common language for sharing information with a Network Management System (NMS).

Network Management Systems
A Network Management System (NMS) is a software tool used to monitor, manage, and optimize the performance, configuration, and security of computer networks. NMSs provide centralized management by monitoring network devices like routers, switches, and servers, and they offer real-time performance data, fault detection, and alerting. NMSs use protocols like SNMP and NetFlow to collect data. Popular NMS softwares include SolarWinds Network Performance Monitor, Nagios, PRTG Network Monitor, Zabbix, and ManageEngine OpManager.

Main components of SNMP

1. SNMP agent

The SNMP agent is the networking device (like a router or a switch) that has SNMP enabled.

  • The SNMP agent has multiple Objects that can be interacted with, for example: a router might have objects for its name, uptime, interfaces and routing table.
  • Each object is assigned an Object Identifier (OID), which is essentially a sequence of numbers used to identify that object.
  • These OIDs are stored in a file called a Management Information Base (MIB).
  • The MIB follows a tree structure where each level in the hierarchy represents a more specific category, leading down to individual data points (or "managed objects") that describe various aspects of a device.

2. The NMS

The Network Management System is a software that accepts and processes SNMP messages.

SNMP usually runs over UDP port 161

MIBs and OIDs

This is an MIB "Management Information Base" where multiple objects are organized into a tree structure. The OID (Object ID) for the object ip for example is .1.3.6.1.1.1.4

Example Scenario

  1. We have a router which we want to configure such that it becomes an SNMP agent, which will enable it to communicate with our NMS.
  2. Inside the router's console, we do the following: Router(config)# snmp-server, which enables the SNMP server on the agent.
  3. We then set the community string, which is similar to a password. If the agent receives a command (from the NMS) with the correct community string, the agent will respond to it.
  4. We can then configure our NMS to use the community string we've set on the agent so that it can send GET or SET requests, which will enable us to manage the agent.
