2.4 Integer Representation

Note that by default, an integral data type will represent signed values (including both negative and positive values within its available range).

Integers are typically stored in 4 consecutive bytes after being converted to binary. 

25 in decimal gets stored as 00000000 00000000 00000000 00011001

Representing Negative Values

Two’s Complement is a method for encoding negative numbers in binary. Imagine we’re using 8 bits to store integers. These 8 bits can represent up to 2^8 different values, covering the range from 0 to 255. In Two’s Complement, the leftmost bit (the most significant bit) is treated as -128 instead of 128.

This scheme allows us to represent numbers in the range -128 up to 127: 

1000 0000 is -128
1000 0001 is -127

...

1111 1111 is  -1
0000 0000 is   0
0000 0001 is   1

...

0111 1111 is 127
