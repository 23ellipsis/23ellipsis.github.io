2.3 Representing Strings and Code

Representing Strings

Consider the following C code:



This code creates a string with the value "12345", and then prints the bytes stored in the memory address that is storing this string. Notice how the value of "12345" is actually stored in memory as 31 32 33 34 35 00. These are the corresponding ASCII encodings for each character (for example, the string "1" corresponds to "31" in ASCII). Also, notice the 00 at the end, which is the null terminator marking the end of the string.

This illustrates the difference between how strings and integers are handled in memory. If we were to store the integer 1 instead of the string "1" (denoted by the quotes), it would be stored in memory as 0x01, not 0x31.

Representing Code

As we know, programs are compiled into binary code that is executed by the CPU. Compilers generate this binary code based on different factors such as CPU architecture and operating system conventions, which is why it can differ across systems.
