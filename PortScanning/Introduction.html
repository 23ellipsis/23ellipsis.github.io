Introduction

Post Scanning is the process of probing target ports to determine which are open and accepting incoming connections.

What exactly are ports?

A port in this context is simply a logical endpoint on a device. (we aren't referring to physical ports where cables connect).

Ports allow multiple services to accept connections at once, having traffic routed to the appropriate service once it reaches the destination. For example, I could run an HTTP Apache service that accepts incoming HTTP requests on my device, and at the same time run an FTP service that accepts incoming FTP connections. Once HTTP or FTP traffic reaches my device (using my IP address), we need to have a way to forward HTTP traffic to the HTTP service, and FTP traffic to the FTP service. This is where we use Port Numbers (embedded in the TCP header) to specify where the traffic should go after it reaches the target device.

Well Known Port Numbers

Some services run on a fixed, well known port number by default. Here is a list:

Port Protocol Service Description
80 TCP HTTP Web traffic
443 TCP HTTPS Secure web traffic
21 TCP FTP File transfer
22 TCP SSH / SFTP Secure shell and file transfer
69 UDP TFTP Trivial file transfer
23 TCP Telnet Unencrypted remote login
3389 TCP RDP Remote desktop
25 TCP SMTP Send mail
110 TCP POP3 Retrieve mail
143 TCP IMAP Mail sync
445 TCP SMB Windows file sharing
2049 TCP/UDP NFS Network file system
53 TCP/UDP DNS Name resolution
111 TCP/UDP RPCbind RPC endpoint mapping
161/162 UDP SNMP Network management

Types of scans

  1. Passive scanning, which includes:
    • Analysing the network traffic using tools as tcpdump and wireshark.
    • Using arp tables.
  2. Active scanning, which includes:
    • Using tools such as nmap, hping, scapy, ping, and tracert.
