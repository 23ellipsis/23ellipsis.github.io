Port States

When you scan a network with tools like nmap, ports can appear in different states, which describe how the port is behaving or responding.

How are port scans actually carried out?

Technically speaking, the process of port scanning includes sending certain packets to a port on a target system and analyzing the response.

The simplest scan type is a TCP connect scan which involves attempting a TCP three-way handshake with the target (on the port we're probing).



Note that this is not the only way to scan a target port. There are more complex scans that involve different types of packets, and there are different techniques for bypassing firewall restrictions.

Port states

  1. Open
    • The port is accessible and responsive.
    • A service is actively listening on that port.

  2. Closed
    • The port is accessible and responsive.
    • Nothing is filtering the port (like a firewall).
    • No service is actively listening on that port.
    • It returns a RST or ICMP unreachable message.
    An ICMP unreachable packet might be returned when the scanning tool sends an ICMP ping request before attempting the actual scan.

  3. Filtered
    • The scanner detects the port is being blocked by a firewall or filter.
    • Usually, the target sends back an ICMP unreachable message or drops the packet in a way that indicates filtering.
    • The port might be open or closed.

  4. Open | Filtered
    • The scanner can't determine whether the port is open or filtered.
    • Usually, the scanner receives no response at all.
    • The port might be filtered by a security system or the service isn't responding.
    • This is common with UDP scans where the target doesn't reply to a UDP packet at all.
