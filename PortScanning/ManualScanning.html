Manual Scanning

TCP Scanning

As discussed earlier, the simplest TCP port scanning technique is referred to as Connect Scanning.

It basically checks if the target can complete a TCP three way handshake (SYN, SYN-ACK, ACK) on a specific TCP port, and if the handshake is successful, the port is considered open.

TCP Connect Scans with Netcat

Netcat (often abbreviated as nc) is a simple but powerful tool used to read and write data across network connections using TCP or UDP.

We can use the following command for a simple TCP connect scan:

nc -nvv -w 1 -z [IP address] [Port]
  • -n: skip DNS lookups
  • vv: verbose (detailed) output
  • -w 1: connection timeout set to one seconds
  • -z: "Zero I/O mode", meaning, do not send any data (the connection is established for scanning, not for sharing data).

UDP Scanning

Since UDP is stateless and does not involve a three-way handshake, the mechanism behind UDP port scanning is different from TCP:

  • If the target UDP port is open, the the packet will be passed to the application layer and the response received will depend on how the application is programmed to respond to empty packets.
  • If the port is close, the target should respond with an "ICMP port unreachable" message.
Remember
When scanning UDP ports, a closed port usually responds with an ICMP "port unreachable" message. If a firewall blocks this response, the scanner cannot distinguish between an open port and a filtered (blocked) one, so it may report the state as "open|filtered." For TCP scans, closed ports normally reply with a TCP RST instead, while filtered ports often produce no response.

UDP Connect Scans with Netcat

nc -nvv -w 1 -z -u [IP address] [Port]

// -u: specifies UDP
